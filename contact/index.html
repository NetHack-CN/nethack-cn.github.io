有疑问？

如果是游戏机制/没那么严肃的问题，欢迎直接来这里问。

上不去GitHub？那就发个E-mail。

想提交bug？

GitHub Issues用于处理从bug报告到长期构想等方方面面的内容。因此，遵循一些简单的规则能让整个流程变得更加顺畅。

Rule zero

请尽可能为issue起一个有意义的标题，因为这是大家首先看到的内容。

Bug报告

在提交错误报告之前，请务必先搜索问题列表，确认该问题是否已被报告过。

您的bug报告应该包括：

  • 您是在哪个操作系统（Windows、Linux、macOS等）上遇到这个bug的
  • 使用的是什么界面（tty、curses、tiles）
  • 版本号（在游戏里按V后弹出的那一长串）
  • bug产生时的情况（最好能够稳定复现，如果不能也至少写一下之前你干了什么）
  • 这里本应发生什么

您的bug报告可以包括：

  • 录屏/截图/终端文本导出（有些内容通过图片展示更直观）
  • 发生bug时的存档文件，不确定是哪个就把存档目录打包过来
